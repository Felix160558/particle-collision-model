# Contributing

欢迎参与修改这个教学模型项目。请优先保持项目简单、可发布、科学表达准确。

## 修改原则

- 保持静态网页结构，不引入必须构建后才能运行的流程。
- 所有站内链接使用相对路径，例如 `./models/example.html` 或 `../src/main.js`。
- 不把临时预览地址、个人本机路径或调试端口写进页面链接。
- 修改模型时，同时检查对应的标题、入口说明、公式说明和交互反馈。
- 新增模型时，把 HTML 页面放在 `models/`，把逻辑模块放在 `src/modules/`。

## 科学表达

- 不为了视觉效果改变物理含义。
- 区分演示模型、近似模型和真实物理定律。
- 公式、单位、变量名要前后一致。
- 如果某个动画只是帮助理解，说明它是可视化表示，不把它写成严格证明。
- 固液气三态变化模型要强调粒子排列与运动方式的变化，避免把单一演示误写成所有物质的完整相图。

## 视觉与交互

- 首页保持清晰的模型入口，不堆放过多说明。
- 模型页优先展示可交互的物理现象。
- 文字不能遮挡核心图形或控件。
- 移动端和桌面端都要能读清标题、按钮和公式。

## 提交流程

1. 修改前先阅读 `./README.md` 和 `./docs/codex-guide.md`。
2. 保持改动范围集中，一次提交只解决一个明确问题。
3. 检查所有新增链接是否为相对路径。
4. 检查浏览器控制台是否有明显报错。
5. 在说明中写清楚改动影响了哪个模型或页面。
